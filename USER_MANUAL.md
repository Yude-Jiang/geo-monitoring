# ST GEO Monitoring Hub — 用户手册

## 第一部分：产品介绍

### 1. 这是什么？

GEO（生成式引擎优化）监测仪表盘。它自动向 Kimi、豆包、DeepSeek、通义千问、文心一言等 AI 平台发送预设提问，用 DeepSeek 模型对 AI 回复进行结构化评估，量化 STMicroelectronics 品牌在各大 AI 平台上的可见度、推荐排名、情感偏好和竞品声量。

### 2. 核心功能

- **多平台自动化巡检** — 支持 5 个主流中文 AI 平台
- **深度归因分析** — 品牌提及率、首推率、主张命中、竞品 SOV、情感评分
- **Prompt 矩阵管理** — 每人可配置自己的监测关键词和意图
- **证据链回放** — 每次监测保留 AI 原始回复文本 + 网页截图

### 3. 多人使用

每个团队成员注册独立账号，各自的监测策略和历史数据互不干扰。数据保留 6 个月。

## 第二部分：操作指南

### 1. 首次使用

**第 1 步 — 打开系统**

浏览器访问系统地址（本地开发默认 `http://localhost:8080`）。

**第 2 步 — 注册或登录**

首次使用点「注册」，填写：

| 字段 | 说明 |
|------|------|
| 用户名 | 英文字母或数字，用于登录 |
| 显示名 | 中文名，在侧边栏和页面上显示 |
| 密码 | 至少 4 位 |

已有账号直接点「登录」输入用户名和密码。

**第 3 步 — 首次爬取（Chrome 登录）**

首次执行监测时，如果 `.env` 中 `HEADLESS=false`，会弹出 Chrome 窗口。系统会依次打开各 AI 平台——**如果某个平台显示登录页面，请手动扫码或输入账号登录一次**。登录后 Chrome 的 session 会被保存到 `.chrome-data/` 目录，后续监测自动复用，不再需要登录。

> 如果部署在服务器上（HEADLESS=true），需先在本地完成上述登录流程，然后把 `.chrome-data/` 目录上传到服务器。

**第 4 步 — 配置你的关键词**

进入「Prompt 矩阵」→ 点右上角「新增监测策略」，输入：

- **Prompt**：你想对 AI 提的问题（如"物联网设备选什么 MCU 最好"）
- **意图**：这条提问属于什么场景（产品发现 / 竞品对比 / 技术咨询 / 品牌口碑）
- **频率**：标记用（目前手动触发，后续可接入定时调度）

保存后，点策略卡片上的「执行监测」即可用该 Prompt 发起一次抓取。

### 2. 各页面说明

#### 2.1 仪表盘 (Dashboard)

系统首页，展示你的品牌监测核心指标：

| 指标 | 含义 |
|------|------|
| **ACS 分数** | 归因置信度综合评分（0-100）。样本量 < 5 时显示"数据不足"，5-19 条显示分数但标注"低置信度" |
| **品牌提及可见度** | AI 回复中提及 ST 品牌的比例 |
| **首选推荐率** | ST 被作为第一推荐品牌的占比 |
| **核心主张命中率** | 回复中关联到预设营销主张的比例 |
| **平均情感分** | LLM 对回复情感倾向的评分（1-10） |

下方还有 7 天可见度趋势图、平台可见度对比柱状图、最近监测记录表。

> 样本量不足时会显示黄色或红色提示条，告诉你当前数据可信程度。

#### 2.2 监测记录 (Observations)

所有历史监测的卡片列表。每张卡片显示：

- 平台标签（Kimi / 豆包 / DeepSeek 等）
- 监测意图、Prompt 文本
- 品牌是否被提及、是否首推、排名、情感分
- 截图缩略图（模拟数据会标注 Simulation）

点击「回放证据」查看完整详情：归因结果、主张命中、AI 原始回复文本、网页截图。

#### 2.3 深度分析 (Analytics)

6 个分析模块：

| 模块 | 说明 |
|------|------|
| **Platform × Intent 热力图** | 不同平台在不同意图下的可见度交叉分析，自动从数据中提取平台和意图 |
| **Benchmark vs Post-launch** | 早期数据与近期数据的四维对比（提及率、首推率、主张命中、情感分） |
| **归因因子贡献度** | 品牌提及、首位推荐、主张命中、来源引用各自的贡献占比 |
| **Competitor SOV** | 竞品声量饼图（TI、NXP、Renesas 等），自动从 AI 回复中提取 |
| **AI 平台表现** | 各平台的可见度和情感均分排行 |
| **策略有效性 Top 5** | 最有效的 5 条 Prompt 策略 |

#### 2.4 Prompt 矩阵 (Prompts)

管理你的监测策略：

- **新增策略**：右上角按钮，填写 Prompt、选意图和频率
- **执行监测**：点策略卡片上的按钮，用该 Prompt 立即执行一次抓取 + 评估
- **全矩阵扫描**：右上角按钮，依次执行所有已配置的策略
- **删除策略**：点卡片上的删除按钮

### 3. 常见问题

**Q: 为什么截图显示"Simulation"而不是真实截图？**

部分 AI 平台有严格的登录墙或反爬机制。当抓取失败时，系统自动降级为模拟数据以保证流程不中断，这些数据会被标记为 Simulation。

**Q: 全矩阵扫描时看不到进度？**

当前版本按顺序逐个执行。Dashboard 会在每个任务完成后自动刷新，请稍等片刻后查看监测记录。

**Q: ACS 分数什么时候才会显示？**

样本量 ≥ 5 条开始显示数值（标注"低置信度"），≥ 20 条后进入正常置信度。

**Q: 数据存多久？**

默认保留 6 个月。超过 6 个月的监测记录会被自动清理。

**Q: 如何添加新同事？**

让同事在登录页点「注册」创建自己的账号即可。每个人的策略和数据互相独立。

**Q: 忘记密码怎么办？**

联系管理员在服务器上通过 SQLite 命令行重置密码，或直接注册新账号。

**Q: 可以在手机上使用吗？**

界面做了移动端适配，手机上可以正常浏览和操作。
